## What changed

- **`evidence=` metadata key** on the ledger's `fw:` line — a pointer to what
  proved a learning: a test name, a `command → result`, a PR/run id, the green
  check. One more `key=value` on the existing single-line comment; omissible.
- **`/flywheel:compound`** (`skills/compound/SKILL.md`) now records `evidence=`
  on every new entry, and — for a genuinely cross-cutting lesson with no single
  proof — writes `evidence=unverified` **explicitly** rather than omitting it,
  so "written on reasoning" is visible, never silent. This makes structural the
  v0.21.0 prose rule ("only compound what this cycle proved").
- **Trust surfaced at consumption**: `scripts/session-start.sh` reads `evidence`
  and prepends an `[unverified — written on reasoning, not proven]` marker to
  entries tagged `evidence=unverified`; `/flywheel:recall`'s cheap list shows
  each entry's evidence pointer / `unverified` flag. So a reader always weighs a
  learning by whether it was proven.
- **Backward compatible by design**: an *absent* `evidence=` means *legacy*
  (pre-convention), which is **not** flagged — only an *explicit*
  `evidence=unverified` is. `evidence` is an additional key the awk reads;
  entries without it parse, score, and inject exactly as before. Relevance
  scoring is unchanged — trust is *shown*, never *ranked* (a wrong-but-relevant
  entry still surfaces, flagged, rather than being hidden).
- **Tests**: `scripts/test-session-start.sh` gains three assertions — an
  `evidence=unverified` entry is flagged; a legacy entry (no `evidence=`) is
  not; a verified entry is not.
- **Docs**: README's ledger section and `/flywheel:help` describe the key and
  the trust signal.
- Advisory in this release: nothing *refuses* to write an entry (a hard
  compound-lint is a deferred follow-up — a blocking gate risks suppressing real
  cross-cutting lessons that resist a single test). Marketplace installs pick
  this up on `/plugin update` + reload; vendored repos on re-vendor — hence
  `requires-action: false`.
