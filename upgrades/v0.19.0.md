## What changed

- **No more whole-ledger reads** — `loop` (step 0), `spec` and `process`
  (prior-art steps) told Claude to `Read` all of `LEARNINGS.md` (~18k tokens
  at 200 entries, twice per loop cycle) on top of the relevance-scored
  SessionStart injection built precisely to avoid that. All three now prime
  from the injected subset and pull specifics with `/flywheel:recall`.
- **Diff-routed review** (`skills/review/SKILL.md`) — a routing step before
  dispatch: docs/comment-only diff → correctness alone; under ~20 changed
  lines → one combined-lens reviewer; security only when the diff touches
  input/auth/secrets/dependencies; performance only for loops/queries/I/O/data
  volume; substantial cross-domain diffs keep the full three-way dispatch.
  Skipped reviewers are always named with the rule that skipped them — no
  silent caps. (Pattern proven manually across three release cycles first:
  ~63–70k tokens per routed review vs ~380k for the audit fan-out.)
- **Evaluator wording honesty** — README and `/flywheel:help` described the
  autoloop evaluator as firing on every keep/discard; aligned to autoloop's
  actual contract (ambiguous results + the final stop decision only).
- **Descriptions on a diet** — `process`, `run`, `autoloop`, `debug`
  frontmatter descriptions (345–431 chars, loaded into every session of every
  repo) trimmed to trigger-conditions ≤ 300 chars; rationale stays in the
  bodies, which load on invocation.
- **Size-capped injection** (`scripts/session-start.sh`) — the SessionStart
  budget was count-based (top 12) with unbounded bodies; injected entries are
  now truncated at ~500 chars with a `[truncated — /flywheel:recall pulls the
  full entry]` tail. New assertion in `scripts/test-session-start.sh`.
- No new commands, agents, or state-format changes; GATEs and fixed contracts
  untouched. Marketplace installs pick this up on `/plugin update` + reload;
  vendored repos on the next re-vendor — hence `requires-action: false`.
