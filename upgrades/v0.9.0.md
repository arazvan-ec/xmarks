## What changed

- **Model routing by agent role.** Following the Claude Code guidance to route
  routine/mechanical work to a faster, cheaper model and reserve capable models
  for judgment calls:
  - `agents/verifier.md` now runs on **`haiku`** — it runs commands and reports
    evidence/exit codes (mechanical), so a fast model fits.
  - `agents/reviewer-correctness.md`, `reviewer-security.md`,
    `reviewer-performance.md` **deliberately stay on `sonnet`** — adversarial
    review is judgment work.
- No new commands, skills, or agents; no config or ledger changes. Vendored and
  marketplace installs pick up the new model on the next re-vendor / reload — no
  action required in installed repos.
- **Overriding:** edit an agent's `model:` frontmatter (e.g. raise the verifier
  to `sonnet`, or a reviewer to `opus` for high-stakes reviews), or set
  `CLAUDE_CODE_SUBAGENT_MODEL` to override all subagents at once. If your org's
  model allowlist excludes `haiku`, Claude Code falls back to the inherited
  model, so this is safe by default.
- **Note on the verifier:** verification is flywheel's core guarantee. Haiku
  handles the mechanical run-and-report; if you ever observe a rationalized
  false-green, raise the verifier back to `sonnet` (a one-line frontmatter change).
