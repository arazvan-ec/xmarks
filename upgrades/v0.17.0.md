## What changed

- **BSD/macOS-safe installer** (`scripts/install-vendored.sh`) — the vendoring
  rename used a GNU-only `sed "0,/re/"` address that dies under
  `set -euo pipefail` on macOS/BSD sed, breaking the documented local install
  path on Macs. Now `1,/re/` (equivalent here: line 1 is always the
  frontmatter `---`, never `name:`).
- **Upgrade pruning** — re-installing a newer version now removes files the
  old version vendored but the new one no longer ships (old→new manifest
  diff; `.pre-flywheel` backups are restored, removals are logged, emptied
  dirs cleaned). Previously such files were orphaned forever and even
  `--uninstall` couldn't see them. The **`--auto-update` choice is sticky**: a
  plain re-install no longer drops the auto-update workflow from the manifest
  (the previous behavior that made it unremovable).
- **Uninstall symmetry for skills** — a pre-existing user skill dir named
  `flywheel-*` had its `SKILL.md` backed up on install, but uninstall deleted
  the backup with the vendored dirs. Uninstall now restores `SKILL.md` from
  its `.pre-flywheel` backup and keeps the dir (agents already behaved this
  way).
- **Generic reviewer agents** — `reviewer-security` and `reviewer-performance`
  carried hardcoded context from this repo's previous life as a bookmarking
  tool ("X/Twitter cookies/session tokens", "per-bookmark DB writes"); since
  flywheel installs these agents into every target repo, that misdirected
  reviews everywhere. Replaced with generic examples.
- **`allowed-tools` honesty** — `compound` now declares `Bash(date *)` (its
  body requires `date +%F`); `ship` declares `Bash(gh *)` (its body offers the
  gh CLI for the PR step).
- **Tests** (`scripts/test-install-vendored.sh`) — three new assertion groups:
  a static no-GNU-sed check, the pruning + sticky-workflow scenario, and the
  skill backup/restore round-trip.
- No skill-body, hook, or state-format changes. Vendored repos pick the fixes
  up on the next re-vendor / auto-update PR; marketplace installs on
  `/plugin update` + reload — hence `requires-action: false`.
