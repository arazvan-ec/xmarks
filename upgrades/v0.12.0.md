## What changed

- **P4 (goal evaluator) — decided against, recorded as skipped.** Assessed
  whether autoloop needs a separate cheap evaluator agent (mirroring `/goal`'s
  Haiku evaluator) to judge its stop condition. Decision: **skip** — `/goal`'s
  evaluator exists because it has no other way to check a condition (it only
  reads the transcript); autoloop already forces a **deterministic metric
  command** to run and its actual output to be recorded every iteration, which
  is strictly stronger evidence than a read-only model judging the same
  transcript. Adding an evaluator on top would not catch anything the metric
  command itself doesn't already prove, so it was redundant work. See the
  decision log in `docs/research/improvement-proposals.md` and
  `docs/research/journal.md`.
- **P5 (token-usage discipline), shipped:**
  - `skills/autoloop/SKILL.md` — new "Token discipline" section: the iteration
    budget is treated as a hard stop (default 10, never unbounded); recommends
    piloting on a small budget (3-5 iterations) before scaling; points to
    `/usage` for actual spend; and gives guidance on when to reach for `/goal`,
    `/loop`, or a dynamic workflow instead of autoloop.
  - `skills/help/SKILL.md` — new bullet under "Good to know" surfacing
    `/usage`, `/goal`, and `/workflows` for spend visibility, plus
    pilot-before-scaling / interval-matching advice.
  - `README.md` — one-line pointer to the new guidance next to the existing
    model-routing note.
- **Version note:** this went through two rebases at merge time — the exact
  collision `docs/research/briefs/README.md` warned about, since P2/P3/P4/P5/P6
  were all built as parallel async briefs off the same base. Originally cut as
  v0.10.0; P2 (git-native memory) merged first and took that number, so this
  moved to v0.11.0; then P3 (read-priming hook) also merged first and took
  *that* number, so this lands as **v0.12.0**. No functional change from either
  rebase — only the version number and the release note's own file name moved.
- No new commands, skills, agents, or config surface. Vendored and marketplace
  installs pick up the updated skill text on the next re-vendor / reload — no
  action required in installed repos.
