## What changed

- `install-vendored.sh --auto-update` now prints the target repo's exact
  `https://github.com/<owner>/<repo>/settings/actions` URL (derived from `origin`) with the
  precise toggle to check, instead of a generic pointer.
- The reusable update workflow writes the same URL to the job summary when opening the PR
  fails, so a missing permission diagnoses itself.
- No behavior change in already-vendored repos: the reusable workflow is referenced
  `@main`, so callers pick this up automatically, and the vendored files are unaffected.
