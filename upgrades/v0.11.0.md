## What changed

- **New read-priming hook** (`scripts/read-prime.sh`), wired as a `PreToolUse`
  hook on `Read` in `hooks/hooks.json`. Before a file read, it greps the
  ledger's typed `files=` metadata (added in v0.10.0) for the target path and,
  if any entry names it, prints a short note listing the matching
  learning(s) — cheap prior context before an expensive read.
- **Advisory only** — the read always proceeds; this never blocks like
  claude-mem's File Read Gate does. Fast and fail-open: no ledger, no match,
  malformed hook input, or no `python3` on `PATH` all produce no output and
  exit 0.
- **New runtime dependency: `python3`**, needed to parse the hook's JSON input
  safely. This was already a build-time dependency of `install-vendored.sh`
  (it uses `python3` to merge `settings.json`), but `session-start.sh` and
  `gate.sh` never needed it at runtime — `read-prime.sh` is the first hook
  script that does. It degrades silently (no output, exit 0) if `python3` is
  missing, so it can never break a session that lacks it.
- `scripts/install-vendored.sh` now also vendors `read-prime.sh` into
  `.claude/flywheel/bin/` and merges a `PreToolUse`/`Read` hook entry into the
  target's `.claude/settings.json` (idempotent, matched by command string, same
  as the existing `SessionStart`/`Stop` hooks); `--uninstall` removes it the
  same way. `scripts/test-install-vendored.sh` asserts it is vendored,
  executable, and merged with the right matcher.
- New `scripts/test-read-prime.sh` (wired into CI) covers matching/non-matching
  paths (relative and absolute), a missing ledger, malformed hook input, and
  the no-`python3` fallback.
- **Vendored installs**: re-run `install-vendored.sh` (or `/flywheel:update`)
  to pick up the new hook script and settings entry — no other action needed.
  Marketplace installs pick it up on `/plugin update` / reload.
