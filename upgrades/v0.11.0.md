## What changed

- **New `evaluator` agent** (`agents/evaluator.md`, `model: haiku`, read-only
  tools). Given the autoloop metric command, target, and the working agent's
  claimed score, it **re-runs the metric command itself** and returns
  `CONTINUE` / `STOP — <reason>`. This mirrors `/goal`'s separate-evaluator
  mechanism, but goes further: `/goal`'s evaluator only reads the transcript,
  while this one independently re-executes the metric command — a genuine
  cross-check against the working agent misreporting or rationalizing its own
  score, not a redundant restatement of flywheel's existing metric-command
  check.
- **`skills/autoloop/SKILL.md`** now dispatches the `evaluator` before keeping/
  discarding an ambiguous iteration or declaring the target met, and logs its
  verdict alongside each iteration. The existing max-iterations budget is
  unchanged — the evaluator adds a check, not a new stop path.
- **Token-usage discipline (P5)**, folded into the same release since both
  touch `autoloop`: explicit guidance to pilot a new metric on a small
  iteration budget before scaling up, match the metric-check interval to how
  fast the change can plausibly move it, state stall criteria up front, and
  use `/usage`, `/goal` status, or `/workflows` to watch spend mid-run.
  Referenced from `skills/autoloop/SKILL.md` and `skills/help/SKILL.md`.
- README documents the new `evaluator` agent and the autoloop self-check.
- No config or ledger changes; no new commands. Vendored and marketplace
  installs pick up the new agent and skill text on the next re-vendor/reload —
  no action required in installed repos.
