## What changed

- **Reopens P4** (previously deferred in v0.12.0 as redundant against autoloop's
  existing metric-command check). That decision assessed a `/goal`-style
  evaluator that only re-reads the transcript — which would indeed add nothing,
  since it can't see anything the metric command's own output hasn't already
  proven. This release builds a **different mechanism**: the evaluator
  **independently re-executes the metric command itself**, rather than reading
  what the working agent reported. That closes a distinct gap the original
  decision named as its own revisit trigger: the working agent both makes a
  change and grades it, so a misreport, a stale/no-op metric command, or an
  honest misreading of the output can slip through uncaught.
- **New `evaluator` agent** (`agents/evaluator.md`, `model: haiku`, read-only
  tools). Given the autoloop metric command, target, and the working agent's
  claimed score, it re-runs the command and returns `CONTINUE` /
  `STOP — <reason>`.
- **`skills/autoloop/SKILL.md`** now dispatches the `evaluator` before keeping/
  discarding an ambiguous iteration or declaring the target met, and logs its
  verdict alongside each iteration. The existing max-iterations budget and the
  v0.12.0 token-discipline guidance are unchanged — the evaluator adds a check,
  not a new stop path or a new spend category.
- README and `/flywheel:help` document the new `evaluator` agent and the
  autoloop self-check.
- No config or ledger changes; no new commands. Vendored and marketplace
  installs pick up the new agent and skill text on the next re-vendor/reload —
  no action required in installed repos.
