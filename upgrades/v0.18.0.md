## What changed

- **Read-priming actually works now** (`scripts/read-prime.sh`; flow-audit
  finding C1). PreToolUse stdout on exit 0 is shown only in transcript mode —
  the "prior learnings touch this file" note never reached the model, in
  marketplace and vendored installs alike. Matches are now emitted as
  `{"hookSpecificOutput": {"hookEventName": "PreToolUse", "additionalContext": …}}`
  (built with `json.dumps`, titles + dates only — no entry bodies); every
  no-match and failure path stays silent exit 0, exactly as before.
- **Bash pre-filter** — the hook fires on every Read and ~all of them have no
  matching entry; a `grep -qF` of the target's basename against the ledger now
  skips the python spawn for that majority. If the path can't be extracted
  cheaply, it falls through to python (correctness over speed, never the
  reverse).
- **session-start scoring hardened** (`scripts/session-start.sh`):
  - The awk metadata parser only looked at the first line after a `## `
    header, so one blank line before `<!-- fw: … -->` silently zeroed that
    entry's relevance (and disagreed with read-prime, which searches the whole
    entry). Blank lines are now skipped.
  - The O(n²) ranking sort (hook-timeout death near ~10k entries) is now a
    bounded top-K selection — only the INJECT_N printed slots get ordered.
  - `date -d '-30 days'` is GNU-only; on macOS the recency signal was
    permanently dead. Added the `date -v-30d` fallback.
  - The remote-version check (a blocking ~650ms curl per session start on
    vendored installs) is cached for the day in `${TMPDIR:-/tmp}` — never the
    working tree; unreadable/unwritable stamp degrades to today's behavior.
- **Tests**: new `scripts/test-session-start.sh` (fixture ledger: blank-line
  tolerance, files > recency > stale ranking, INJECT_N budget + tail,
  no-ledger path) wired into CI; `scripts/test-read-prime.sh` now asserts the
  JSON envelope on matches and silence everywhere else.
- **Docs**: README's read-priming section describes the injection truthfully.
- No skill, agent, or state-format changes. Vendored repos pick both hook
  scripts up on the next re-vendor / auto-update PR; marketplace installs on
  `/plugin update` + reload — hence `requires-action: false`.
