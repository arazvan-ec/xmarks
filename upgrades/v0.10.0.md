## What changed

Git-native memory, first release (P2 of the improvement backlog — see
`docs/research/git-native-memory-design.md`). Goal: claude-mem-style
selectivity for `.claude/flywheel/LEARNINGS.md`, without a database or worker.

- **Typed entries.** `/flywheel:compound` now writes one entry per learning as
  `## <type>: <title>` (type: `decision`/`gotcha`/`pattern`/`bugfix`) followed
  by a single greppable `<!-- fw: type=…; date=…; files=…; spec=…; pr=…;
  branch=… -->` metadata line, then the prose body. `type` and `date` are
  required; `files`/`spec`/`pr`/`branch` are optional but drive relevance.
- **Relevance-scored, budgeted injection.** `scripts/session-start.sh` no
  longer dumps the first ~50 lines. It now scores every ledger entry against
  the current session (branch, changed files, active spec, recency) and
  injects only the top `FLYWHEEL_LEARNINGS_INJECT` full entries (default 12),
  plus a one-line pointer for the rest. Scoring: `+3` if an entry's `files`
  overlaps the changed files, `+2` if its `branch` or `spec` matches the
  current one, `+1` if it's from the last ~30 days.
- **New `/flywheel:recall <query>`.** Progressive disclosure for anything not
  auto-injected: lists matching entries cheaply (title + metadata), then
  expands one on request. Added to the README command table and
  `/flywheel:help`.
- **Backward-compatible.** Old free-prose entries (no metadata line) still
  load — they're always-eligible but low-priority, with recency recovered
  from the old `## YYYY-MM-DD — …` header when possible. Nothing in the
  existing ledger needs to be rewritten.

## Not in this release

Deferred by the locked design: a `.tsv` metadata index (grep-live is fine at
current ledger sizes), semi-auto Stop-hook staging, and claude-mem/Engram
interop. The read-priming `PreToolUse` hook (P3) is a separate, follow-on
release that depends on this one's typed `files=` metadata.
