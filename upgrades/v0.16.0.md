## What changed

- **Live progress obligation, both pillars** (roadmap item P16; piloted as
  `flow-audit` contract v3 + the run #001 report). The owner should never have
  to ask "how is it going": progress is a visible ledger plus a live report,
  and chat stays reserved for gates, blockers, and the final summary.
- **`/flywheel:run`** (`skills/run/SKILL.md`) — new step 0 *Progress ledger*:
  one host-task per contract Rule, states updated at every transition; a
  telemetry report at `.claude/flywheel/runs/<slug>/<date>.html` (ledger +
  timings, gates, unit telemetry, outputs, verdict — never secrets),
  regenerated per transition and republished to a stable artifact URL.
  Fail-open: reporting never blocks the run.
- **`/flywheel:process`** (`skills/process/SKILL.md`) — the contract template
  gains a **Progress reporting** section, so every new contract inherits the
  obligation.
- **`/flywheel:loop`** (`skills/loop/SKILL.md`) — the six phases become visible
  tasks with per-gate state updates and a cycle report at
  `.claude/flywheel/runs/<spec-slug>/<date>.html`.
- **`/flywheel:work`** (`skills/work/SKILL.md`) — each plan task becomes a
  visible task, flipped the moment its local check goes green (never in bulk
  afterwards); inside a loop cycle it also updates the cycle report.
- **Docs:** README gains a "Live progress" note and the `runs/` state line;
  `/flywheel:help` documents the feature and its state list now names
  `processes/`, `DATA.md`, and `runs/` alongside `specs/` and the ledger.
- **No new skills, agents, hooks, or scripts; no config or ledger-format
  changes.** Prompt-only additions to four existing skills plus docs — vendored
  repos pick it up on re-vendor, marketplace installs on the next
  `/plugin update` + reload. Existing contracts keep working; they gain the
  Progress reporting section on their next deliberate revision. Nothing to
  migrate — hence `requires-action: false`.
