## What changed

- **New `fixture` learning type** (alongside `decision`/`gotcha`/`pattern`/
  `bugfix`). A `fixture` entry records *how to set up the world* — the recipe to
  build a valid stub/fixture for a domain entity, seed the datastore, or stand
  up a test harness, plus the fields/relationships that are easy to get wrong
  (its title names the entity, e.g. `fixture: editorial stub for homeTag`). This
  is what a session most expensively re-derives; compounding it means the next
  one starts from the recipe.
- **`/flywheel:compound`** (`skills/compound/SKILL.md`) — `fixture` added to the
  type list, with an explicit close-of-cycle prompt ("did we discover how to
  build a stub / seed data / stand up a harness the next cycle shouldn't
  re-derive?") and a rule to record *how to build* test data, never real
  credentials or PII.
- **`/flywheel:work`** (`skills/work/SKILL.md`) — primes from `fixture` entries
  before building an entity's test data, and gains an advisory capture trigger
  alongside its delegation thresholds: ~2+ non-trivial stub files or ~5 tool
  calls spent standing up test data → *offer* to record a `fixture` entry.
  Advisory only — it never forces or blocks the loop.
- **`/flywheel:spec`** (`skills/spec/SKILL.md`) — the prime step folds any
  matching `fixture` recipe into the spec, so planning starts with it.
- **`/flywheel:process`** (`skills/process/SKILL.md`) — pillar-2 contracts pull
  `fixture` recipes for the entities they read/write, tied to DATA.md.
- **Docs**: README's ledger/state section and `/flywheel:help` name the type.
- **No plumbing change.** The memory pillar already carries an arbitrary
  `type=`, so the SessionStart relevance scorer (`scripts/session-start.sh`),
  the read-priming hook, and `/flywheel:recall` handle `fixture` entries with
  zero changes — the new type flows through unchanged. Existing entries and the
  four existing types are untouched. Marketplace installs pick this up on
  `/plugin update` + reload; vendored repos on re-vendor — hence
  `requires-action: false`.
